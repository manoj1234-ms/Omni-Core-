## 🏛️ v3.2.2 Architecture: "The Verification Nexus"

### 🛡️ 1. Omni-Guard: The Core Insertion Point
A simple, high-performance wrapper designed to protect your LLM outputs. It integrates seamlessly into any application.

```python
from omni_guard import OmniGuard

guard = OmniGuard()

# llm_output = llm.generate("Tell me a fact about space")
```

## 🏗️ The 6-Step Explicit Grounding Pipeline

1.  **Claim Decomposition**: LLM output is split into testable propositional claims.
2.  **Deterministic Validation**: RegEx-based checks for dates, numbers, and tech specs.
3.  **Global Retrieval**: Parallel search across **Supabase L3 Collective Memory** and **Live Web Search** (DDGS).
4.  **Source Ranking**: Scoring based on domain trust weights (.gov, .edu, arxiv.org).
5.  **Iterative Consensus**: Weighted voting between swarm agents, measuring **Conflict Gravity** to detect disagreements.
6.  **Structured Explanation**: Returns a machine-readable JSON object with confidence breakdown and source links.

---

## 🧬 Key Features (Phases 1-3)

### 📈 Adaptive Swarm Trust (Cognitive Credit Score)
Agents are dynamically vetted based on their historical accuracy.
- **Performance Tracking**: Grounding success boosts trust (+0.05).
- **Trust Decay**: Inactive agents experience decay to ensure the Hive stays accurate.
- **Sliding Consensus**: Weighted voting favors agents with higher historical credibility.

### 🕸️ Cognitive Knowledge Graph (Memory Evolution)
Moving from flat logs to linked intelligence.
- **Relational Reasoning**: Retrieves `Subject -> Relation -> Object` triplets to provide deeper contextual grounding.
- **Recency Trust**: Prioritizes newer verified facts while archiving old ones.

### 🌊 Supabase L3 Collective Knowledge
Uses **Supabase (PostgreSQL)** as its permanent **Global Collective Memory Pool**.
- **Cross-Agent Knowledge**: New logic learned by one agent is instantly available to the entire swarm.
- **Session Persistence**: Maintains task state and context across autonomous sessions.

---

## 🚀 Grounding Modes
- **`strict`**: Rejects and blocks response if confidence < 0.7.
- **`warn`**: Returns verdict with warning markers for low-confidence claims.
- **`auto_fix`**: Attempts to reconstruct the response using top-scored evidence chunks.
